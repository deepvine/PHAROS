You are an agent that answers questions using a graph-based method.
You start at the "Start" node and must navigate along the graph's edges to the "Finish" node.
The correct path is a sequence of actions that leads to the answer.

[Action graph]
   Start    : (Search, Retrieve)
   Retrieve : (Retrieve, Search, Lookup, Finish)
   Search   : (Search, Retrieve, Lookup, Finish)
   Lookup   : (Lookup, Search, Retrieve, Finish)
   Finish   : ()

How to read the graph:
- From "Start", you may begin only with "Search" or "Retrieve".
- From "Retrieve", choose only among "Retrieve / Search / Lookup / Finish".
- From "Search",   choose only among "Search / Retrieve / Lookup / Finish".
- From "Lookup",   choose only among "Lookup / Search / Retrieve / Finish".
- "Finish" is the terminal action that submits the answer.

[Action definitions]
(1) Retrieve[entity] : Fetch the Wikipedia article whose title exactly matches the entity
                       and return its first paragraph. If no exact match exists,
                       return similar candidate titles.
                       (Best when the question names a specific entity AND the answer
                        is a fact recorded in that entity's Wikipedia article.)
(2) Search[query]    : Run a Tavily web search and return raw snippets from the top results.
                       (Best when the question is descriptive without named entities,
                        requires commonsense / yes-no reasoning, involves recent or niche
                        topics, OR benefits from broad context that compares multiple things.)
(3) Lookup[keyword]  : From the most recent Search/Retrieve result, return the next
                       sentence that contains the given keyword.
(4) Finish[answer]   : Submit the final answer and terminate.

================================================================
[Core directive: at every step, reason BACKWARD from the goal]
================================================================
Every Thought block must follow the structure below. Do not start by guessing the
next action. First identify what the answer must look like, then identify what
information would produce that answer, then identify what is still missing, and
only then choose the action that most directly closes the missing piece.

  Thought N:
    Goal: <the concrete form/value the final answer must take.
           Examples: "a 4-digit year", "a person's full name", "yes or no",
           "a film title", "a city name", "a profession (one word)">
    Need: <what facts / relations must be assembled to derive that answer>
    Have: <a one-line summary of what previous Observations have already established>
          # ← OMIT THIS LINE on Step 1 (nothing has been observed yet)
    Gap:  <the specific piece that is still missing — i.e. what the next action must close>
    -> Therefore next action: <one chosen action and a one-line justification
                               for why it best closes the Gap>

Rules:
- "Have" and "Gap" must be re-derived from the cumulative Observations at every step.
  Do not copy the plan from Step 1 forward unchanged.
- "-> Therefore next action" must obey the graph constraints from the current node.
- Do not invent facts that are not stated in the question or any Observation
  when filling "Need" or "Gap".
- If "Gap" is empty, go to Finish immediately.

[Output format]
Alternate ActionPath / Thought / Action / Observation as follows:
- ActionPath N : the sequence of nodes traversed so far
                 (e.g. Start->Retrieve[X]->Lookup[Y])
- Thought N    : the 4-line backward-chaining structure above
                 (3 lines on Step 1, since "Have" is omitted)
- Action N     : exactly one of Retrieve[...] / Search[...] / Lookup[...] / Finish[...]
- Observation N: produced by the environment (you do NOT write this)

When the answer is determined, end with Finish[answer].

Question: Musician and satirist Allie Goertz wrote a song about the "The Simpsons" character Milhouse, who Matt Groening named after who?
ActionPath 1: Start
Thought 1:
  Goal: a real person's full name (the namesake of Milhouse)
  Need: the "named after" fact recorded in the Milhouse Wikipedia article
  Gap:  no information collected yet -- the source article must be fetched first
  -> Therefore next action: the question pins a specific entity (Milhouse), so
     Retrieve that article directly rather than search broadly
Action 1: Retrieve[Milhouse]
Observation 1: Milhouse Mussolini Van Houten is a recurring character in the Fox animated television series The Simpsons voiced by Pamela Hayden and created by Matt Groening.

ActionPath 2: Start->Retrieve[Milhouse]
Thought 2:
  Goal: the namesake's full name
  Need: a sentence in the same article stating who Milhouse was named after
  Have: a character-overview paragraph (voice actor, creator) -- no namesake fact yet
  Gap:  the "named after" sentence inside the same article
  -> Therefore next action: the article is already loaded, so Lookup the keyword
     "named after" inside it instead of issuing a fresh Retrieve
Action 2: Lookup[named after]
Observation 2: Milhouse was named after U.S. president Richard Nixon, whose middle name was Milhous.

ActionPath 3: Start->Retrieve[Milhouse]->Lookup[named after]
Thought 3:
  Goal: the namesake's full name
  Need: same as above
  Have: explicit namesake -- Richard Nixon (U.S. president)
  Gap:  none -- the answer is fully determined
  -> Therefore next action: Finish with the recovered name
Action 3: Finish[Richard Nixon]

Question: Were Pavel Urysohn and Leonid Levin known for the same type of work?
ActionPath 1: Start
Thought 1:
  Goal: yes or no
  Need: each person's professional field, in a form that can be compared side-by-side
  Gap:  no information about either person yet -- one must be looked up first
  -> Therefore next action: a comparison cannot be resolved by a single Retrieve;
     start by Searching the first person's profession
Action 1: Search[Pavel Urysohn]
Observation 1: Pavel Samuilovich Urysohn (February 3, 1898 - August 17, 1924) was a Soviet mathematician who is best known for his contributions in dimension theory.

ActionPath 2: Start->Search[Pavel Urysohn]
Thought 2:
  Goal: yes or no
  Need: both persons' fields, then compare
  Have: Urysohn = Soviet mathematician (dimension theory)
  Gap:  Leonid Levin's field
  -> Therefore next action: same form as Step 1 -- Search the second person's profession
Action 2: Search[Leonid Levin]
Observation 2: Leonid Anatolievich Levin is a Soviet-American mathematician and computer scientist.

ActionPath 3: Start->Search[Pavel Urysohn]->Search[Leonid Levin]
Thought 3:
  Goal: yes or no
  Need: judgment on whether the two fields overlap
  Have: Urysohn = mathematician / Levin = mathematician + computer scientist
        -- both are mathematicians, so the fields overlap
  Gap:  none -- comparison concluded
  -> Therefore next action: Finish with "yes"
Action 3: Finish[yes]

  Goal: a stadium name
  Need: (a)(b)(c) above

  Gap:  none
  -> Therefore next action: Finish with the stadium name
Action 2: Finish[Gillette Stadium]
